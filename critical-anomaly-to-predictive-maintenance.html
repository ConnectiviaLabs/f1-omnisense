Critical Anomaly → Predictive Maintenance

How a critical severity anomaly triggers feature importance extraction, forecasts the top features, generates insights, and produces an autonomous maintenance schedule.

1
Anomaly Detection & Severity Classification
agents/detection/anomaly_analyzer.py

The AnomalyAnalyzer agent continuously processes data streams. When an anomaly is detected, the confidence score determines its severity. Only Critical and High severity anomalies proceed to feature importance extraction.

Severity Confidence Threshold Triggers Feature Extraction?
CRITICAL ≥ 0.96 Yes — full SHAP analysis
HIGH 0.80 – 0.94 Yes — full SHAP analysis
MEDIUM 0.55 – 0.78 No — skipped
LOW < 0.55 No — skipped
confidence ≥ 0.96
2
Feature Importance Extraction (SHAP)
anomaly_analyzer.py → _populate_feature_contributions()

For critical anomalies, TreeSHAP runs on an IsolationForest fit of the last 200 data points. Each numeric feature gets an importance score (0–1). The top 5 features are retained. Execution time: <1 second.

Method
TreeSHAP on IsolationForest
Fallback
Z-score proxy (if data < 200 points)
Features Kept
Top 5 by |importance|
Execution Time
< 1 second

3
Critical Anomaly Event Published
agent_event_bus.py → Redis pub/sub channel: agent.events

The anomaly analyzer publishes a critical_anomaly event carrying the feature importance payload and candidate features list. This event fans out to 5 subscribers including the Predictive Maintenance Agent.

// Event payload published to Redis { "type": "critical_anomaly", "severity": "critical", "anomaly_score": 0.97, "confidence": 0.98, "collection": "sensor_readings", "feature_importance": { "top_features": [ { "feature": "temp_sensor", "importance": 0.8524 }, { "feature": "pressure", "importance": 0.6451 }, { "feature": "vibration", "importance": 0.4123 } ], "source": "anomaly_analyzer" }, "candidate_features": ["temp_sensor", "pressure", "vibration"], "immediate_action_required": true }

response/predictive_maintenance_agent.py → _extract_candidate_features()

The PM agent extracts candidate features from the event payload. Features are selected by priority: anomaly-detected features first, then config targets, then auto-detected numeric columns. Only the top 3 are forecasted.

AI Insight Generation (LLM Synthesis)
services/kex/KeXnStore.py → InsightGenerator

For each forecasted feature, the LLM generates a natural-language insight combining the forecast trend, metrics, and domain context. These insights become the human-readable explanation attached to each maintenance task.

LLM Provider
Groq (qwen3-32b) or Ollama
Input
Feature + forecast + metrics + thresholds

Priority Condition Response Window
