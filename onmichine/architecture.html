Overview

What It Is

A ReAct-style agentic loop where a Groq-hosted LLM (openai/gpt-oss-120b) autonomously orchestrates a tabular ML pipeline. The agent decides which tools to call and in what order, inspects intermediate results, and adapts strategy — like an AI data scientist.

What It Produces

A fully reproducible model bundle: fitted model + preprocessor + schema + leaderboard + model card + reproducibility lock. Supports regression, binary classification, and multiclass classification with automatic task inference.

Design Principles

Deterministic ML — the LLM reasons about strategy, never touches math. Strict artifact contracts — every run produces auditable outputs. Graceful fallback — if Groq is down, runs a sequential deterministic pipeline automatically.

Quick Start

from agentml import run, RunConfig

result = run(RunConfig(
    data_path="data.csv",
    target_column="Speed",
))
print(result.best_trial.model_name)
print(result.holdout_metrics)
Agent Loop Architecture

ReAct Multi-Turn Loop

The Groq LLM receives tool results after each call and decides the next action. Up to 15 turns. Falls back to sequential if no Groq API key or on failure.

Task Inference Rules
## Deterministic inference from target column properties

if dtype == bool                          → binary_classification
if dtype == string and unique == 2        → binary_classification
if dtype == string and unique > 2         → multiclass_classification
if dtype == numeric and unique == 2       → binary_classification
if dtype == numeric and unique ≤ 20      → multiclass_classification
else                                       → regression

## Metrics
regression       → primary: RMSE      secondary: MAE, R²
binary           → primary: AUC-ROC   secondary: F1, Accuracy, LogLoss
multiclass       → primary: F1-weighted secondary: Accuracy, LogLoss
Data-Aware Model Strategy

Small Dataset (<500 rows)

Skip deep ensembles. Baselines + HistGB only. Short HPO budget. No voting ensemble.

Many Categoricals

Low-cardinality (≤15) → OneHotEncoder. High-cardinality → OrdinalEncoder. CatBoost preferred when available.

Imbalanced Classes

Stratified train/test split. Falls back to non-stratified if rare classes cause errors. AUC-ROC as primary metric.
